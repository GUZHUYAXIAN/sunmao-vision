## 1. 这个项目到底在做什么？

**一句话**：把货物塞进集装箱，让空间利用率尽可能高，然后在浏览器里用 3D 画面展示出来。

**稍微展开**：

1. 用户导入工程图纸 (PDF)，系统自动生成 3D 模型 (OBJ 文件)
2. 用户手动校准模型的真实尺寸（因为 PDF 转换出来的模型尺寸不准）
3. 用户配置集装箱的规格和货物的属性（重量、数量等）
4. **求解器**计算最优堆叠方案 + 扎带固定方案
5. 前端用 **Three.js** 把结果渲染成 3D 场景，用户可以旋转、缩放、点选查看
6. 导出 Excel 装箱清单

### 关键依赖关系

```
contracts ← 被 solver 和 web 共同依赖
solver    ← 被 web 调用
web       ← 面向用户的最终应用

规则：
  ✅ contracts 不依赖任何其他内部包
  ✅ solver 只依赖 contracts
  ✅ web 可以依赖 contracts 和 solver
  ❌ solver 绝不能依赖 web（不能 import React / Three.js）
```

### 3.2 移植指南

如果你想帮忙将旧逻辑移植到新架构，请遵循以下步骤：

1. **先读新契约** — 打开 [`doc/architecture/02-data-contracts.md`](./architecture/02-data-contracts.md)，理解新的数据类型定义。旧代码中的类型需要适配到新的 Zod Schema。

2. **先写测试** — 在新架构的对应包的 `__tests__/` 目录下先写好测试用例。确保测试失败（因为还没有实现）。

3. **提取核心逻辑** — 从旧文件中提取算法核心，去掉 UI 相关代码和旧的类型定义，使用新的 `@sunmao/contracts` 类型。

4. **让测试通过** — 实现逻辑直到所有测试通过。

5. **提交 PR** — 附上旧文件路径和新文件路径的对照说明，方便维护者审查。

**示例 commit message**:

```
feat(solver): migrate collision detection from legacy geometry.ts

Extracted AABB collision detection logic from:
  legacy_code/20260314-2/SmartContainer-3D/src/lib/geometry.ts

Adapted to use @sunmao/contracts types (CargoInstance, Container).
Added unit tests for edge cases (touching faces, nested items).
```
