## 1. 整体页面架构

前端采用**两阶段**设计：设置流程（步骤引导）+ 主工作台（3D 交互）。

### 1.1 阶段一：设置流程（WorkflowStepper）

四步引导式流程，类似有限元分析的前处理步骤：

```text
┌─── Step 1 ───┐  ┌─── Step 2 ───┐  ┌─── Step 3 ────┐  ┌── Step 4 ──┐
│   导入模型    │→│   尺寸标定    │→│  配置集装箱    │→│  提交求解   │
│  (PDF → OBJ)  │  │  (缩放校准)   │  │  + 货物参数    │  │             │
│               │  │              │  │  + 扎带选择    │  │             │
└──────────────┘  └──────────────┘  └───────────────┘  └────────────┘
```

| 步骤 | 组件 | 输入 | 输出 |
| --- | --- | --- | --- |
| Step 1: 导入模型 | `ImportStep` | PDF 文件 | ModelAsset[] |
| Step 2: 尺寸标定 | `CalibrateStep` | 模型 + 实际尺寸 | ScaleRecord[] |
| Step 3: 配置参数 | `ConfigureStep` | 集装箱规格 + 货物属性 + 扎带选型 | SolveRequest |
| Step 4: 提交求解 | `SolveStep` | SolveRequest | SolveResult |

求解完成后自动进入阶段二。

## 2. 七大交互功能规范

### 2.1 旋转拖动（Camera Controls）

**组件**: `CameraControls.tsx`

基于 Three.js `OrbitControls` 的自定义扩展：

| 操作 | 功能 | 实现方式 |
| --- | --- | --- |
| 滚轮上下 | 放大/缩小 | `OrbitControls.enableZoom` |
| 按住滚轮拖拽 | 旋转视角 | `OrbitControls` 中键绑定 |
| Ctrl + 滚轮拖拽 | 平移视角 | 自定义事件：检测 `ctrlKey` + 中键 |
| Shift + 滚轮拖拽 | 缩放（替代滚轮） | 自定义事件：检测 `shiftKey` + 中键 |
| V + 左键点击 | 重设旋转中心点 | 射线检测点击位置，更新 `controls.target` |

**LOD 自动切换**：镜头距离变化时根据阈值自动切换模型精度（远粗近细）。

### 2.2 高亮显示（Selection Highlighting）

**组件**: `useSelection.ts` + `Viewport3D.tsx`

```text
选中效果：
┌───────────────────────────────┐
│  选中的物体                     │
│  - opacity = 1.0 (完全不透明)   │
│  - 边缘发光 (OutlinePass)      │
│  - emissive 色调增亮           │
├───────────────────────────────┤
│  其他所有物体                   │
│  - opacity ≤ 0.4 (至少降低60%) │
│  - material.transparent = true │
│  - 视觉上明显"退后"            │
├───────────────────────────────┤
│  取消选中后                     │
│  - 所有物体恢复 opacity = 1.0  │
│  - 移除发光效果                │
└───────────────────────────────┘
```

**双向联动**：

- 左侧树选中 → 3D 场景对应物体高亮 + 其余半透明
- 3D 场景点击物体 → 左侧树自动定位并高亮对应节点

### 2.3 菜单折叠（Tree Panel）

**组件**: `TreePanel.tsx`

树形结构，交互模式模仿 Windows 资源管理器：

```text
▾ 集装箱 1（20尺标准柜）             ← 母菜单，可折叠/展开
  ├── 发动机壳体 #1                  ← 子项
  ├── 发动机壳体 #2
  └─▾ 机架总成 #1                   ← 大件货物（内含子菜单）
      ├── 控制盒 #3                 ← 嵌套：小件在大件内部
      └── 控制盒 #4
▾ 集装箱 2（40尺高柜）
  ├── ...
▾ 暂存区                            ← 未入柜的货物
  ├── 散热片 #1
  └── 散热片 #2
```

**完全双向拖拽逻辑**：

| 操作方向 | 行为 |
| --- | --- |
| 树 → 3D | 在树中拖拽小框到大框节点下 → 3D 场景中小框自动移入大框内部 |
| 3D → 树 | 在 3D 场景中把小框拖进大框里 → 树菜单自动同步节点移动 |
| 树/3D 剪切 | Ctrl+X 剪切后粘贴到另一个容器 → 两侧同步更新 |

**核心原则**: 树和 3D 场景共享同一份 `instances` 状态（单一数据源），任何一侧的操作都更新这份状态，两侧自动重新渲染。

### 2.4 选中逻辑（Selection Logic）

**组件**: `useSelection.ts`

完全模仿 Windows 资源管理器的选中行为：

| 操作 | 行为 |
| --- | --- |
| 单击 | 选中该项，取消其他所有选中 |
| Ctrl + 单击 | 切换单个项目的选中状态（可多选不连续项） |
| Shift + 单击 | 范围选中（从上一个选中项到当前项之间的全部） |
| Shift 跨容器 | 范围跨越集装箱边界：第一箱选到末尾 + 第二箱从开头选到当前项 |
| Ctrl + A | 全选当前容器内所有项 |
| Delete | 删除选中项（移入暂存区） |
| Ctrl + Z | 撤销上一步操作 |
| Ctrl + X | 剪切选中项到暂存区 |

### 2.5 重量标尺（Weight Color Scale）

**组件**: `WeightRuler.tsx` + `useWeightScale.ts`

底栏显示渐变色条，将货物重量映射为颜色：

```text
🔴 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 🔵
最重 (红色)                        最轻 (天蓝色)

上限：自动取所有货物中最大重量，向上取整到 10 的倍数
下限：自动取所有货物中最小重量，向下取整到 10 的倍数
```

**色阶等级自动判定**：

| 货物种类数 | 色阶等级数 |
| --- | --- |
| ≤ 10 种 | 10 阶 |
| ≤ 15 种 | 15 阶 |
| > 15 种 | 20 阶 |

**自定义选项**：

- 用户可修改上限/下限值
- 用户可切换配色方案（红-蓝、红-黄-绿、热力图等）
- 提供"还原默认"按钮

### 2.6 重量显示（Weight Display）

**组件**: `WeightToggle.tsx`

底栏右侧两个独立切换按钮：

```text
[净重 ●]  [毛重 ○]

净重 = 纯货物重量之和
毛重 = 货物 + 扎带 + 集装箱皮重
```

| 状态 | 显示内容 |
| --- | --- |
| 仅开启净重 | 每个集装箱下方显示纯货物总重 |
| 仅开启毛重 | 每个集装箱下方显示毛重（含皮重） |
| 同时开启 | 同时显示两个数值 |

**悬停交互**：

- 鼠标悬停在左侧树的货物节点 → 3D 视口浮窗显示该货物重量
- 鼠标悬停在集装箱母菜单 → 浮窗显示该集装箱的净重和毛重

## 4. 状态管理架构

### 4.1 核心状态

```typescript
interface AppState {
  // 项目数据
  project: Project;

  // 选中状态
  selectedIds: Set<string>;        // 当前选中的货物实例 ID
  lastSelectedId: string | null;   // 最后一个选中的 ID（用于 Shift 范围选）

  // 视图状态
  weightDisplay: {
    showNet: boolean;              // 是否显示净重
    showGross: boolean;            // 是否显示毛重
  };
  weightScale: {
    min: number;                   // 色阶下限
    max: number;                   // 色阶上限
    levels: 10 | 15 | 20;         // 色阶等级
    colorScheme: string;           // 配色方案
  };

  // 编辑状态
  editingItemId: string | null;    // 正在编辑的货物 ID
  undoStack: Action[];             // 撤销栈
  redoStack: Action[];             // 重做栈

  // 求解状态
  solveStatus: "idle" | "solving" | "done" | "error";
}
```

### 4.2 状态管理方案

- **轻量级响应式**: 使用 React Context + useReducer
- **不引入重量级状态库**: 本项目状态结构清晰、不涉及跨页面共享，无需 Redux / Zustand
- **单一数据源**: 树面板和 3D 视口共享同一份状态，保证双向联动

### 6.2 异常反馈策略

| 阶段 | 反馈方式 | 说明 |
| --- | --- | --- |
| 初期 | 提示式警告 | Toast 通知，不阻断用户操作 |
| 成熟期 | 阻断式报错 | Modal 弹窗，必须处理后才能继续 |

警告信息统一收集在 `SolveResult.warnings` 中，前端根据 `severity` 决定显示方式。
